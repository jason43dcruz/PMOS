# Skill: Draft Stakeholder Communication

**Trigger:** "Draft update for [person]," "Slack to the team," "Email [stakeholder]"

When the user asks you to draft a stakeholder communication:

1. Read skills/write-like-me.md (apply this skill to all output)
2. Read Knowledge/product-context.md to understand audience priorities
3. Read Knowledge/writing-style.md for voice and tone
4. Read relevant task files for data and context
5. Calibrate for the audience:
   - Manager: Focus on blocks, impact, lovability, tables (customize per Knowledge/product-context.md)
   - Engineering: Technical precision, tradeoffs
   - Leadership: Headline metrics, strategic alignment
6. Format for the right channel (Slack = short, email = structured, Confluence = detailed)

Output should sound like the user, not a chatbot. Use their phrases. Avoid their banned words.
