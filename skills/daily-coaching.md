# Skill: Daily Coaching

**Trigger:** User says "what should I work on today", "clear my backlog", or asks for daily focus and priorities.

---

## Daily Focus (3 Tasks Max)

When the user asks what to work on today:

1. **Read GOALS.md** — Understand the current priorities and strategic direction.
2. **Scan Tasks/** — Check all active task files for status, deadlines, and blockers.
3. **Check for blocked items** — Identify anything waiting on decisions, external input, or dependencies.
4. **Recommend three focus tasks**, ordered by:
   - **Shipping deadline first** (soonest ship date wins)
   - **User impact second** (higher impact if tied)
   - **Strategic alignment third** (core to goals if still tied)
5. **Assess workload** — If the day looks too heavy, say so directly and suggest what to defer or defer to later in the sprint.

Keep it tight: one sentence per task, why it matters, what "done" looks like.

---

## Backlog Clearing

When the user says "clear my backlog" or asks to clean up BACKLOG.md:

1. **Read BACKLOG.md** — Get the full list of open items.
2. **Check for duplicates** — Scan Tasks/ to make sure we're not creating redundant work.
3. **Ask for clarity** on anything ambiguous:
   - Priority (P0/P1/P2/P3)?
   - Timeline (this sprint, next sprint, later)?
   - Which goal does it connect to (or does it)?
   - Is it blocked on anything?
4. **Create task files** with proper structure:
   - Goal alignment (link to GOALS.md)
   - Priority level
   - Category (e.g., strategy, execution, analysis, coaching)
   - Concrete next action (not vague; something you can do right now)
5. **Empty the backlog** — Once everything has a home in Tasks/, delete or archive BACKLOG.md.

If an item doesn't tie to a goal and the user can't justify it, flag it and move on. Don't force-fit things.

---

## Goal Alignment

Every coaching conversation starts and ends with GOALS.md.

- **Always tie work back to goals.** If something doesn't connect, ask "why does this matter?" — don't assume.
- **Challenge assumptions** when something doesn't add up. If the user is planning something that contradicts a goal, push back.
- **Push back on over-planning** — if we're filling the day with low-impact work, say so. "This is a 10-hour day for what looks like 4 hours of impact. What should we cut?"
- **Use the goal doc as a forcing function.** If we're debating priority and GOALS.md is silent, that's a sign we need to revisit goals first.

---

## Operating Rules

- **One question at a time** when sparring on priorities or open items. Ask one, get an answer, move to the next. This keeps decisions crisp.
- **Be concrete.** Suggest specific next steps, not frameworks. "Call the sales team Thursday" beats "align with sales."
- **Bias to shipping.** If something is 70% clear and ready to move, move it. We can course-correct.
- **Keep the day sharp.** Three focus tasks is the default. If the user naturally has more or fewer, that's fine — but push back if it's obviously overloaded.
